# Usage Templates

这份模板给项目成员直接复制调用 `project-issue-journal` 用。

## 什么时候用

适合在下面这些时机主动调用：

1. 做完一个功能或一个 change，准备复盘时。
2. 遇到一个问题排查了很久，怀疑它不只是表面 bug 时。
3. 同一类问题已经重复出现两次以上时。
4. 你感觉“这件事以后面试、写总结、写文章一定会讲到”时。
5. 你发现自己在重复做某类手工工作，但还没想清楚根因时。

## 最小调用模板

```text
使用 project-issue-journal，整理这次开发里我遇到的问题，并更新 docs/tem/项目开发问题记录.md
```

## 找隐性关键点

```text
使用 project-issue-journal，帮我找出这次开发里我没有完全意识到、但会显著影响后续决策和工作的关键点，并更新 docs/tem/项目开发问题记录.md
```

## 梳理重复性工作

```text
使用 project-issue-journal，梳理这次开发里我所有重复性的工作内容，并说明这些重复工作背后暴露出的流程、边界、自动化或文档问题，更新 docs/tem/项目开发问题记录.md
```

## 功能开发后复盘

```text
使用 project-issue-journal，复盘这次功能开发过程。重点记录：
1. 我遇到的高价值问题
2. 我没意识到但影响很大的关键点
3. 我重复做了哪些事情
并更新 docs/tem/项目开发问题记录.md
```

## Bug 排查后复盘

```text
使用 project-issue-journal，总结这次 bug 排查过程，不要只记录表面症状，要重点记录根因、隐性关键点和重复性工作，并更新 docs/tem/项目开发问题记录.md
```

## Change 收口后复盘

```text
使用 project-issue-journal，基于这次 change 的实现和收口过程，补充项目开发问题记录。重点看决策问题、架构问题、管理问题和协作问题。
```

## 面试 / 总结素材提炼

```text
使用 project-issue-journal，回看最近这次开发，提炼那些适合在面试、复盘或文章里展开讲的问题，并更新 docs/tem/项目开发问题记录.md
```

## 输出偏好模板

如果你想指定偏好，可以在模板后面继续补要求。

### 偏向隐藏问题

```text
使用 project-issue-journal，少写表面问题，重点写我没意识到的高影响关键点，并更新 docs/tem/项目开发问题记录.md
```

### 偏向管理和流程问题

```text
使用 project-issue-journal，重点梳理这次开发中的流程问题、协作问题和重复性劳动，并更新 docs/tem/项目开发问题记录.md
```

### 偏向文章素材

```text
使用 project-issue-journal，把这次开发里最值得扩展成文章的话题整理出来，写进 docs/tem/项目开发问题记录.md
```
