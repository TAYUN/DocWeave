# DocWeave 应用壳层布局规格

本文档定义 DocWeave 前端应用壳层的 UI 规格，包括 `AppHeader`、Sidebar 模式和主内容区框架。

它服务的是应用壳层设计，不负责路由拆分和代码目录组织。相关实现边界见 [`../workflow/frontend-route-architecture.md`](../workflow/frontend-route-architecture.md)。

## 布局边界

- `PublicLayout`：用于 `/login`、`/register`、`/forgot-password`、`/invite/:token` 等公开页；不显示 Sidebar。
- `AppLayout`：用于已登录业务页；负责 `AppHeader + Sidebar + MainContent`。
- `ShareLayout`：用于公开分享页；按分享阅读场景单独设计，不复用工作台 Sidebar。

## AppHeader 组件规格

AppHeader 是所有已登录页面的顶部横栏，承载全局导航和信息。

**布局**：三段式

```text
[左侧]                [中间]               [右侧]
Logo + 品牌名        全局搜索框 ⌘K        新建按钮 + 用户头像下拉
```

| 区块 | 内容 | 说明 |
|------|------|------|
| 左侧 | Logo 图标 + "DocWeave" 品牌名 | 点击回到 `/` |
| 中间 | 搜索框：`<Search size={14} />` + "搜索文档..." + `⌘K` 快捷键提示 | 全局搜索入口，M7 前只能作为未启用占位或说明入口 |
| 右侧 | 新建入口 + 用户头像下拉菜单（个人设置/退出登录） | 只放当前阶段能闭环的动作，不默认加入无实现支撑的入口 |

**关键尺寸**：固定高度 `h-11`（44px），`border-bottom` 与 Sidebar 对齐

**行为规则**：

- Header 始终固定在视口顶部，不随页面内容滚动
- 搜索框在 M7 之前可先不实现真实搜索，但需要明确标注为未启用或即将提供
- 用户头像显示当前用户姓名首字母，点击展开 DropdownMenu：个人设置 / 工作区设置 / 退出登录
- 通知入口不属于当前 change 的必备范围；如果后续加入，必须先补清楚来源、空态和点击去向

## Sidebar 模式

Sidebar 承载页面级导航和上下文信息。根据当前路由切换显示模式。

**通用规格**（所有模式适用）：

- 宽度：`w-56`（224px）
- 背景色比 MainContent 略深（Mantine 默认侧边栏语义）
- 当前选中项高亮（`variant="filled"` vs `variant="light"`）
- 滚动：独立 `overflow-y-auto`，与 MainContent 互不影响

| 模式 | 触发路由 | 显示内容 | 关键行为 |
|------|---------|---------|---------|
| **GlobalMode** | `/` 工作台首页 | 快捷导航按钮（工作台）+ 空间列表入口 + 最近文档列表 | 空间列表可点击进入 `/spaces/:spaceId`，文档条目点击进入 `/documents/:documentId` |
| **SpaceMode** | `/spaces/:spaceId` | 空间切换器（下拉选择其他空间）+ 当前空间内文档树 | 文档树展开/折叠，当前文档高亮 |
| **DocumentMode** | `/documents/:documentId` | 空间面包屑 + 所属空间文档树 + 文档操作（收藏/分享/删除） | 面包屑显示所在的完整路径，文档树中当前文档高亮 |
| 无 Sidebar | PublicLayout 路由 | 不显示 | 页面水平居中，无侧边栏 |

**空间切换器行为**（SpaceMode 特有）：

- 顶部显示当前空间名称和下拉箭头
- 点击展开所有空间列表供切换
- 切换后导航到对应空间的详情页

## 响应式约定

- Desktop：保留完整 Header + Sidebar + MainContent 结构，主任务区维持最高视觉权重。
- Tablet：Sidebar 可收窄或切换为抽屉，但不能挤压主任务表单、列表或编辑器。
- Mobile：单列优先，次级信息后置、折叠或下沉；不把桌面多栏机械堆叠为单列长页。
