# P03 空间详情页

## 路由
`/spaces/:spaceId`

## 用途
展示单个空间的详细信息，列出该空间下的所有文档，并提供在该空间下创建新文档的入口。

## 内容分区

| 区块 | 内容 | 说明 |
|------|------|------|
| 空间信息面板 | 空间名称 + 简介 + 元数据（创建者/文档数/更新时间）+ "新建文档"按钮 | 页面顶部，按钮触发内联表单或 Modal |
| 创建文档表单 | 标题 + 摘要输入 + 创建/取消按钮 | 默认折叠，点击"新建文档"后展开；文档列表为空时自动展开 |
| 文档列表面板 | 该空间下所有文档的列表 | 全宽列表 |

**设计决策**：
- 创建文档表单默认折叠，由按钮触发展开，避免始终占用可视空间
- 文档列表为空时表单自动展开，引导用户创建第一篇文档
- 空间信息面板增加元数据（文档数、最后更新），提供更多上下文

## 状态规格

| 状态 | 表现 |
|------|------|
| 加载中 | 数据随 WorkbenchShell 一起加载 |
| 空间不存在（404） | "未找到对应空间" 错误卡片 |
| restricted / 无权限（403） | "你没有权限访问这个空间" 说明 + 返回工作台路径；页面级受限不伪装成空态 |
| 文档列表为空 | "还没有文档" 空态提示 + "新建文档"按钮（表单自动展开） |
| disabled | "新建文档"按钮在提交中、空间只读或缺少前置条件时保持可见但不可操作，并说明原因 |
| 创建文档成功 | Notification 绿色 + "文档创建成功"，折叠表单 |
| 创建文档失败 | 内联红色 Alert + 错误信息，表单保持展开 |
| 路由参数无效 | TanStack Router 404 处理 |
| 加载空间信息失败 vs 加载文档列表失败 | 分别提供独立的错误提示和重试按钮，而非统一错误卡片 |

## 导航关系

- **从哪来**：
  - 工作台首页点击空间卡片
  - 地址栏直接输入 `/spaces/:spaceId`
- **到哪去**：
  - 点击文档 → `/documents/:documentId`
  - 创建文档 → 刷新当前页面列表
- **错误回退**：空间不存在 → 显示错误卡片，用户可点击导航返回工作台

## 实现约束

- 使用 Mantine 组件：`Paper`, `Stack`, `Group`, `TextInput`, `Textarea`, `Button`, `Title`, `Text`, `Badge`, `Alert`
- API 调用：`listSpaces`, `listDocuments`, `createDocument`, `getSpaceTree`
- 空间信息通过 `spaces` 列表过滤获得（当前实现限制）
- 文档列表通过 `documents` 列表过滤 `spaceId` 获得（当前实现限制）
- 创建文档后 invalidate `documents`, `spaces`, `space-tree` 三个 query 缓存
- 创建文档表单折叠状态通过本地 state 控制：`isCreateFormOpen`，默认为 `false`；文档列表为空时自动设为 `true`
- 后续 M2 改造建议：增加独立的空间详情 API，不依赖全量列表过滤
- 当前空间 ID 为 UUID 格式，应避免直接展示给用户
