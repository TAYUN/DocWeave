# P03 空间详情页

## 路由
`/spaces/:spaceId`

## 用途
展示单个知识空间的标题、简介和文档目录，并提供在当前空间下新建文档的入口。

## 当前实现摘要

当前页面已经不是旧规格里那种“简单信息面板 + 列表”。它更像一个轻量 Hero + 目录页：

- 顶部 Hero 区承载空间名称、说明和主操作
- Hero 内还有一张欢迎说明卡，补足空间语义
- 创建文档表单作为独立区块插在 Hero 和目录之间
- 文档区更明确地强调“目录感”，而不是普通列表页

## 布局（当前实现）

```text
┌──────────────────────────────────────────────────────┐
│ (Sidebar: SpaceMode)                                 │
├──────────────────────────────────────────────────────┤
│ Space Hero                                           │
│ [图标] 空间名称                         [新建文档]    │
│        文档数 / 根文档数                [更多操作]    │
│        空间说明                                        │
│                                                      │
│ 欢迎说明卡                                            │
│                                                      │
│ 创建文档表单（按需展开；空空间时自动展开）            │
│                                                      │
│ 文档目录                                              │
│ 说明文案                                   N 篇       │
│ 文档列表 / 空态 / 错误提示                            │
└──────────────────────────────────────────────────────┘
```

## 内容分区

| 区块 | 内容 | 说明 |
|------|------|------|
| Space Hero | 空间名称、简介、文档统计、主操作 | 页面顶部主区 |
| 欢迎说明卡 | 对知识空间用途的轻解释 | Hero 内的辅助区 |
| 创建文档表单 | 标题、摘要、创建动作 | 默认隐藏；空空间自动展开 |
| 文档目录 | 当前空间下的全部文档 | 按最近更新时间排序 |

## 设计要点

- 页面首先强调“这是一个空间”，其次才是“这里有多少文档”
- 主操作“新建文档”放在 Hero 右侧，符合当前工作流
- 文档目录的语气应像浏览手册目录，而不是浏览交易列表
- 说明卡和列表都保持轻边框、轻背景差，不制造重层级

## 状态规格

| 状态 | 表现 |
|------|------|
| 默认 | 展示 Space Hero、说明卡、文档目录 |
| 空间不存在（404） | “未找到对应空间” 错误卡片 |
| restricted / 无权限（403） | “你没有权限访问这个空间” + 返回工作台 |
| 文档列表为空 | 目录区显示空态，创建文档表单自动展开 |
| 创建文档成功 | Notification 绿色 + 刷新目录 |
| 创建文档失败 | 表单内保留错误提示，表单不自动收起 |
| 文档列表加载失败 | 在目录区显示内联错误 Alert，不打断 Hero |

## 导航关系

- 从哪来：
  - 工作台首页点击空间卡片
  - 地址栏直接输入 `/spaces/:spaceId`
- 到哪去：
  - 点击文档 → `/documents/:documentId`
  - 创建文档成功后留在当前页
- 错误回退：空间不存在或受限时可返回工作台

## 实现约束

- 使用 Mantine 组件：`Paper`, `Stack`, `Group`, `TextInput`, `Textarea`, `Button`, `Title`, `Text`, `Alert`
- 当前空间信息通过 `spaces` 列表过滤得到
- 当前文档目录通过 `documents` 列表过滤 `spaceId` 得到
- 创建文档成功后 invalidate `documents`, `spaces`, `space-tree`
- 顶部“新建文档”动作驱动下方表单展开并滚动到表单位置
- 当前页面不展示 UUID 等实现细节给用户
