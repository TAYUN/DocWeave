# P01 登录页

## 路由

`/login`

## 用途

用户输入邮箱和密码进行身份认证，登录成功后进入工作台。

## 当前实现摘要

当前页面已经不是“顶部品牌区 + 居中单卡片”的旧结构，而是更接近产品化 SaaS 登录页的双区布局：

- 左侧为产品展示区，说明 DocWeave 的协作、AI 和知识组织能力
- 右侧为登录表单区，承载真实登录入口
- 小屏下隐藏左侧展示区，只保留紧凑表单壳

这种实现仍然符合当前设计规范，因为它依旧保持了暖中性底色、轻边框、低装饰和内容优先，只是比旧规格更完整。

## 布局（当前实现）

```text
┌────────────────────────────────────────────────────────────┐
│ 左侧展示区（lg+）                  │ 右侧登录表单区         │
│                                    │                        │
│ DocWeave                           │ 欢迎回来               │
│ Knowledge OS                       │ 登录你的工作区         │
│ 让知识流动，让协作发生。             │                        │
│                                    │ 开发账号（DEV only）    │
│ [实时协同] [AI 问答] [三层结构] ... │ 邮箱                   │
│                                    │ 密码                   │
│                                    │ [登录]                 │
│                                    │ 错误 Alert / 重试       │
│                                    │ 注册页后续接入提示      │
└────────────────────────────────────────────────────────────┘
```

## 内容分区

| 区块         | 内容                         | 说明                                                |
| ------------ | ---------------------------- | --------------------------------------------------- |
| 展示区       | 品牌标识、能力说明、功能卡片 | `lg` 及以上显示，用于建立产品上下文，不承担登录动作 |
| 表单区       | 标题、说明、登录表单         | 页面主任务区                                        |
| 开发账号提示 | 预填账号信息                 | 仅开发环境可见，使用 `import.meta.env.DEV` 控制     |
| 错误提示     | Alert + 可选重试按钮         | 区分凭据错误与网络错误                              |
| 注册提示     | 文字说明                     | 当前阶段只提示“后续接入”，不作为真实跳转入口        |

## 设计要点

- 登录主任务始终集中在右侧表单区，不和展示信息混排
- 展示区可以提供产品气质，但不应演变成营销页
- 小屏优先保护登录动作，展示内容自然退场
- 错误反馈采用普通工作台语义色，不制造高压告警感

## 状态规格

| 状态                | 表现                                                             |
| ------------------- | ---------------------------------------------------------------- |
| 默认                | 展示表单，预填开发账号（仅开发环境）                             |
| 输入中              | 表单正常可编辑                                                   |
| 提交中              | 按钮显示 `loading` + “登录中...”，避免重复提交                   |
| disabled            | 提交中保持按钮可见但不可重复操作                                 |
| 登录失败 - 凭据错误 | 红色 Alert：“邮箱或密码不正确”                                   |
| 登录失败 - 网络错误 | 红色 Alert：“无法连接到服务器，请检查网络” + “重试”按钮          |
| restricted / 无权限 | 不做独立页面态；若服务端返回禁用或拒绝原因，直接显示明确错误文案 |
| 已登录跳转          | 登录成功后跳转到 `/`                                             |

## 导航关系

- 从哪来：地址栏直接进入 `/login`，或鉴权失败后跳转
- 到哪去：登录成功后进入 `/`
- 错误回退：登录失败停留当前页

## 实现约束

- 使用 Mantine 组件：`Paper`, `Stack`, `TextInput`, `PasswordInput`, `Button`, `Title`, `Text`, `Alert`
- API 调用：`apps/web/src/lib/api.ts` 中的 `login`
- Token 存储：`lib/auth.ts` 中的 `saveAccessToken`
- 登录成功后 invalidate `current-user`, `spaces`, `documents`
- 开发账号提示仅在 `import.meta.env.DEV` 下显示
- 当前不提供真实注册入口，只保留后续阶段提示
