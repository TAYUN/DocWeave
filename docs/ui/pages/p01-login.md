# P01 登录页

## 路由
`/login`

## 用途
用户输入邮箱和密码进行身份认证，登录成功后进入工作台。

## 内容分区

| 区块 | 内容 | 说明 |
|------|------|------|
| 品牌区 | DocWeave 标识 + 标语 | 页面顶部 |
| 登录卡片 | 标题 + 说明 + 表单 | 居中 Paper 卡片 |
| 开发账号提示 | 预填账号信息 | 仅开发环境可见，使用 `import.meta.env.DEV` 控制 |
| 邮箱输入 | TextInput | 必填，email 类型 |
| 密码输入 | TextInput | 必填，password 类型 |
| 登录按钮 | Button | 主操作，loading 状态显示"登录中..." |
| 错误提示 | Alert | 区分"凭据错误"与"网络错误"，网络错误附加"重试"按钮 |
| 注册入口 | 文字链接 | 引导新用户注册（P05 注册页规划中，M2 扩展阶段实现） |

## 状态规格

| 状态 | 表现 |
|------|------|
| 默认 | 展示表单，预填开发账号（仅开发环境） |
| 输入中 | 表单正常可编辑 |
| 提交中 | 按钮显示 `loading` + "登录中..."，表单不可提交 |
| disabled | 登录按钮在提交中或必填项未完成时保持可见但不可点击 |
| 登录失败 — 凭据错误 | 红色 Alert：**"邮箱或密码不正确"**，表单可重新提交 |
| 登录失败 — 网络错误 | 红色 Alert：**"无法连接到服务器，请检查网络"** + "重试"按钮 |
| restricted / 无权限 | 登录页本身无页面级无权限状态；若服务端返回账号被禁用等拒绝原因，应以明确错误文案说明，而不是静默失败 |
| 已登录跳转 | token 存在时自动跳转到 `/` |
| 加载中 | 无（登录页不需要异步加载数据） |

## 导航关系

- **从哪来**：地址栏直接输入 `/login`，或 token 过期/无效时重定向
- **到哪去**：登录成功 → `/` 工作台首页
- **错误回退**：登录失败 → 留在当前页
- **注册入口** → `/register`（P05 规划中，当前阶段可先不实现跳转）

## 实现约束

- 使用 Mantine 组件：`Paper`, `Stack`, `TextInput`, `Button`, `Title`, `Text`, `Alert`
- API 调用：`apps/web/src/lib/api.ts` 中的 `login` 函数
- Token 存储：`lib/auth.ts` 中的 `saveAccessToken`
- 登录成功后 invalidate query 缓存：`current-user`, `spaces`, `documents`
- 已有 token 时自动重定向到 `/`
- **开发账号提示**: 使用 `import.meta.env.DEV` 条件渲染，生产环境不显示
- **错误区分**: 根据 API 返回的错误类型（网络错误 vs HTTP 401/403）显示不同的错误提示文案
- 不实现自动注册功能（注册页为独立页面 P05）
