# P04 文档编辑页

## 路由
`/documents/:documentId`

## 用途
用户在此页面查看和编辑单篇文档的标题、摘要和正文内容。这是当前产品最核心的正文工作区。

## 当前实现摘要

当前实现已经不是旧规格中的“双栏编辑器 + 元数据侧栏”。现在的页面是更克制、更内容优先的一栏编辑流：

- 顶部是轻量工具栏，只保留返回路径和状态
- 中间是标题、更新时间、摘要和正文编辑器
- 底部是未保存提示与保存动作

这版实现更符合当前设计规范：弱化 chrome，让标题和正文成为主角。

## 布局（当前实现）

```text
┌──────────────────────────────────────────────────────┐
│ (Sidebar: DocumentMode)                              │
├──────────────────────────────────────────────────────┤
│ Toolbar                                              │
│ [返回空间] > 文档标题                     [状态 Badge] │
│                                                      │
│ 标题输入（弱边框 / 接近纸面标题）                    │
│ 更新时间                                              │
│ 摘要输入                                              │
│                                                      │
│ BlockNote 编辑器                                      │
│                                                      │
│ 未保存提示 Alert（按需出现）                          │
│ [保存文档]                               错误提示      │
└──────────────────────────────────────────────────────┘
```

## 内容分区

| 区块 | 内容 | 说明 |
|------|------|------|
| 顶部工具栏 | 返回空间、当前文档标题、状态 badge | 只保留必要导航和状态 |
| 编辑主区 | 标题、更新时间、摘要、正文 | 页面绝对主区 |
| 保存反馈区 | 未保存提示、保存按钮、错误消息 | 页面底部操作收口 |

## 设计要点

- 页面优先表达“正在写文档”，而不是“正在管理一条记录”
- 顶部工具栏像轻量面包屑，不是强操作台
- 标题输入应有直接在纸面上改标题的感觉
- 状态反馈只做功能性提醒，不用重装饰
- 元数据不再单独占一个右侧面板

## 状态规格

| 状态 | 表现 |
|------|------|
| 加载中 | `LoadingState` |
| 编辑中 | 标题、摘要和编辑器可编辑 |
| disabled | 无改动时保存按钮保持可见但不可操作 |
| restricted / 无权限 | 明确说明当前文档不可编辑，并提供返回路径 |
| 文档不存在（404） | “未找到对应文档” 错误卡片 |
| 有未保存修改 | 显示黄色提示 Alert，并激活 `beforeunload` |
| 保存成功 | Notification 绿色 + 清除未保存状态 |
| 保存失败 | 保留草稿内容，并在底部显示错误消息 |
| 内容格式异常 | `parseDocumentContent` 兜底到默认内容 |

## 导航关系

- 从哪来：
  - 工作台首页点击文档入口
  - 空间详情页点击文档
  - 地址栏直接进入 `/documents/:documentId`
- 到哪去：
  - 顶部返回按钮 → `/spaces/:spaceId`
  - 保存文档后留在当前页
- 错误回退：文档不存在或受限时可返回工作台

## 实现约束

- 使用 Mantine 组件：`Paper`, `Stack`, `Group`, `TextInput`, `Textarea`, `Button`, `Text`, `Badge`, `Alert`
- API 调用：`getDocumentById`, `updateDocument`
- 编辑器组件：`@docweave/editor` 中的 `DocumentEditor`
- 保存内容包含 `title + summary + content`
- 路由切换后应重置编辑状态
- 当前不做自动保存
- 当前不做协同冲突处理
