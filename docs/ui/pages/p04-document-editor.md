# P04 文档编辑页

## 路由
`/documents/:documentId`

## 用途
用户在此页面查看和编辑单篇文档的标题、摘要和正文内容。这是 DocWeave 产品的核心页面。

## 内容分区

| 区块 | 内容 | 说明 |
|------|------|------|
| 状态栏 | 文档状态标签（草稿/已发布/已归档） | 页面顶部，**仅显示状态**，不含标题/摘要 |
| 编辑器面板（左） | 标题输入 + 摘要输入 + BlockNote 编辑器 + 保存按钮 | **主面板**，表单模式，所有编辑操作集中在此 |
| 元数据面板（右） | 创建者 + 所属空间 + 最后更新 + 版本号 | **辅助面板**，xl 及以上显示在右侧，小屏下折叠 |

**设计决策**：
- 标题和摘要的编辑入口**唯一化**：只在编辑器面板中提供输入框，移除顶部状态栏的标题/摘要展示，避免用户混淆"改哪个"
- 元数据面板在 M2 阶段可暂时显示调试信息（文档 ID、空间 ID），但应规划替换为有意义的上下文（创建者、协作者、版本号）
- 未保存修改时在编辑器底部显示"有未保存的修改"提示条

## 状态规格

| 状态 | 表现 |
|------|------|
| 加载中 | Suspense fallback（组件级） |
| 编辑中 | 标题/摘要/编辑器可编辑，保存按钮可用 |
| disabled | 保存按钮在无改动、提交中或当前字段校验未通过时保持可见但不可操作 |
| restricted / 无权限 | 若用户仅有查看权限或文档不可编辑，应明确说明当前为只读/无权限状态，并提供返回路径，不把页面伪装成普通编辑态 |
| 有未保存修改 | 编辑器底部出现黄色提示条："有未保存的修改"，`beforeunload` 事件激活 |
| 保存成功 | Notification 绿色 + "文档已保存"，清除未保存标记 |
| 保存失败 | 内联红色 Alert + 保留草稿内容，未保存标记不清除 |
| 空文档 | 编辑器预填一段"从这里开始编写你的文档" |
| 文档不存在（404） | 错误卡片："未找到对应文档" + "返回工作台"按钮 |
| 文档 ID 变化 | 检测路由参数变化，重置表单状态并清除未保存标记 |
| 内容格式异常 | `parseDocumentContent` 兜底到默认段落 |
| 用户尝试离开 | `beforeunload` 事件弹出确认对话框："你有未保存的修改，确定离开吗？" |

## 导航关系

- **从哪来**：
  - 工作台首页点击文档条目
  - 空间详情页点击文档条目
  - 地址栏直接输入 `/documents/:documentId`
- **到哪去**：
  - 保存文档 → 刷新当前页面
  - 面包屑/导航 → 回到空间详情页或工作台首页
- **错误回退**：文档不存在 → 显示错误卡片 + "返回工作台"按钮

## 实现约束

- 使用 Mantine 组件：`Paper`, `Stack`, `SimpleGrid`, `Group`, `TextInput`, `Textarea`, `Button`, `Title`, `Text`, `Badge`, `Alert`
- API 调用：`getDocumentById`, `updateDocument`
- 编辑器组件：`@docweave/editor` 中的 `DocumentEditor` 组件（封装 `@blocknote/mantine`）
- 内容格式：BlockNote JSON 格式，序列化为字符串存储
- 保存时提交 title + summary + content 三个字段
- 保存成功后更新本地 state，防止表单与后端状态不一致
- 路由切换时重置表单（`useEffect` 监听 `document.id` 变化）
- **未保存修改检测**：通过 `useEffect` 监听 title/summary/content 变化，与初始值比较。有差异时：
  - 激活 `beforeunload` 事件
  - 显示底部提示条
- 不做自动保存（M2 阶段手动保存即可）
- 不做协同冲突处理（M3 阶段再处理）
- 元数据面板当前可显示 ID 等调试信息，但后续应替换为创建者、协作者、版本号等有意义的信息
