# P06 搜索页

## 路由
`/search`

## 用途
为用户提供文档级与片段级搜索入口，帮助快速回到目标文档或相关内容上下文。

## 当前阶段定位

- 当前为占位规格，用于约束后续 M7 RAG 搜索实现
- 在搜索能力未落地前，不应被表现为完整可用页面
- 当前代码库中没有对应已落地页面，本文件不代表当前 UI 已存在

## 最小规格

- 主任务：查找文档或内容片段
- 主内容：搜索输入、结果列表、筛选条件、跳转入口
- 状态要求：未输入、搜索中、无结果、搜索失败、结果可跳转、disabled、restricted / 无权限
- 导航关系：可从 AppHeader 搜索入口或工作台内部入口进入
- 实现边界：优先保护搜索输入与结果列表，不把页面做成介绍页

## 状态要求细化

- `disabled`：搜索输入或筛选控件在请求进行中可暂时不可操作，但应保持可见
- `restricted / 无权限`：若当前用户无权搜索某些范围，应明确说明受限范围，并给出可返回的工作台入口

## 后续补全时必须明确

- 搜索结果粒度是文档级、片段级还是混合
- 结果点击后的跳转位置和高亮策略
- 与 RAG 问答页之间的边界
