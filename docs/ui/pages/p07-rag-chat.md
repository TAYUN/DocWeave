# P07 RAG 问答页

## 路由
`/chat`

## 用途
为用户提供基于知识库内容的问答入口，返回答案同时保留来源与回到文档的路径。

## 当前阶段定位

- 当前为占位规格，用于约束后续 M7 RAG 问答实现
- 在问答能力未落地前，不应被表现为可直接完成任务的真实页面
- 当前代码库中没有对应已落地页面，本文件不代表当前 UI 已存在

## 最小规格

- 主任务：提问并查看答案与引用来源
- 主内容：问题输入区、回答区、来源列表、回到文档入口
- 状态要求：提问中、无答案、失败、来源缺失、disabled、restricted / 无权限
- 导航关系：可从搜索页或全局入口进入，结果应能跳回文档页
- 实现边界：答案展示不能脱离来源链路，避免做成孤立聊天壳

## 状态要求细化

- `disabled`：提问按钮在请求进行中或输入为空时保持可见但不可操作
- `restricted / 无权限`：若当前工作区未开通问答能力或当前用户无权访问，应明确说明原因，并提供返回搜索页或工作台的路径

## 后续补全时必须明确

- 来源引用的最小展示格式
- 与编辑器内 AI 助手的分工边界
- 历史对话是否保留
