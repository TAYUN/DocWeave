# P02 工作台首页

## 路由
`/`

## 用途
用户登录后的落地页，提供空间概览和文档总览。是工作台的核心导航枢纽。

**主任务优先级**：
1. 有空间时 → 默认展示空间列表，用户快速进入已有空间
2. 无空间时 → 引导用户创建第一个空间

## 内容分区

| 区块 | 内容 | 说明 |
|------|------|------|
| 引导区 | 页面标题 + 引导说明 + "新建空间"按钮 | 顶部欢迎区域，按钮为次级操作 |
| 空间列表面板 | 所有空间的列表，点击进入空间详情 | **主区**，视觉权重最高 |
| 文档列表面板 | 所有文档的列表 | **次区**，位于空间列表下方，全宽 |

**设计决策**：
- "创建空间"从独立面板降级为引导区内的一个按钮，避免与空间列表同权重并列
- 点击"新建空间"应触发 Modal 或内联表单，而不是始终展开表单面板
- 空间列表是页面主区，文档列表是次区，用户应先选空间再找文档
- 空态时（无空间列表），引导区应更突出地提示"创建第一个空间"

## 状态规格

| 状态 | 表现 |
|------|------|
| 加载中 | 显示 LoadingState "正在加载工作台" |
| 空间列表为空 | "还没有空间" 空态提示 + "创建第一个空间"主按钮引导 |
| 文档列表为空 | "还没有文档" 空态提示 + 引导进入空间创建文档 |
| 空间有数据但文档仍在加载 | 空间列表正常显示，文档区显示内联 Loader，避免布局跳动 |
| disabled | "新建空间"按钮在提交中、达到创建限制或缺少前置输入时保持可见但不可操作，并说明原因 |
| restricted / 无权限 | 若当前用户可登录但无权查看某些工作台数据，应显示说明与回退路径；不把缺权限数据静默当作空态 |
| 创建空间成功 | Notification 绿色 + "空间创建成功" |
| 创建空间失败 | 内联红色 Alert + 错误信息 |
| API 错误 | 工作台异常卡片 + 重试按钮 |
| 认证错误 | 自动清除 token 并跳转到 `/login` |
| 文档列表超过 50 条 | M2 阶段无分页，但应标注性能风险。后续需引入分页或按空间分组 |

## 导航关系

- **从哪来**：登录成功 → 自动跳转到 `/`
- **到哪去**：
  - 点击空间卡片 → `/spaces/:spaceId`
  - 点击文档条目 → `/documents/:documentId`
  - 退出登录 → `/login`
- **错误回退**：认证失败 → `/login`

## 实现约束

- 使用 Mantine 组件：`Paper`, `Stack`, `SimpleGrid`, `Group`, `Button`, `Title`, `Text`, `Badge`, `Alert`, `Modal`（创建空间用 Modal 触发）
- API 调用：`listSpaces`, `listDocuments`, `createSpace`
- 数据依赖：先加载 `currentUser`，成功后加载 `spaces` 和 `documents`
- 创建空间成功后 invalidate `spaces` query 缓存
- "新建空间"点击后弹出 Modal，而非始终展开表单面板
- 文档列表不做分页（M2 阶段限制），但超过 50 条时应开始考虑引入分页
- 不对文档做"我的"筛选（M2 阶段限制）
- 后续 M2 改造建议：为已注册用户（有空间）和首次使用用户（无空间）设计不同的着陆布局
