# P02 工作台首页

## 路由

`/`

## 用途

用户登录后的落地页，帮助用户从“知识空间”进入文档上下文。这是当前工作台的核心导航枢纽。

## 当前实现摘要

当前页面已经从旧的“空间列表 + 文档列表”两段式首页，演进为更贴近生产力工具的工作台结构：

- 顶部 Hero 区负责欢迎、主操作和关键指标
- 中段是“常用空间”，强调快速回到上下文
- 下段是“我的空间”，展示完整空间概览并带出最近文档

页面重点已经从“把所有文档单独列一遍”，转向“先通过空间组织信息，再进入文档”。

## 布局（当前实现）

```text
┌──────────────────────────────────────────────────────┐
│ (Sidebar: GlobalMode)                                │
├──────────────────────────────────────────────────────┤
│ Hero                                                 │
│ Knowledge Spaces Badge                               │
│ 知识空间                                             │
│ 说明文案                          [新建知识空间]      │
│                                   [打开常用空间]      │
│                       [空间数] [文档总数]            │
│                                                      │
│ 常用空间                                             │
│ 空间概览卡片 × N                                     │
│                                                      │
│ 我的空间                                             │
│ 空间概览卡片 × N，每张卡带最近文档入口               │
└──────────────────────────────────────────────────────┘
```

## 内容分区

| 区块     | 内容                             | 说明                         |
| -------- | -------------------------------- | ---------------------------- |
| Hero 区  | 页面标题、说明、主操作、统计指标 | 首页视觉重心，但仍保持低装饰 |
| 常用空间 | 优先进入的空间入口               | 快速回到最近上下文           |
| 我的空间 | 全量空间概览                     | 页面主信息区                 |

## 设计要点

- 首页主任务是“进入空间”，不是“先浏览全量文档”
- 顶部 Hero 只做轻引导，不做重品牌化营销区
- 卡片重点放在空间及其最近文档，符合三层结构的知识组织方式
- 信息层级依赖标题、留白和轻边框，不依赖重投影和大背景

## 状态规格

| 状态                | 表现                                                 |
| ------------------- | ---------------------------------------------------- |
| 默认                | 展示 Hero、常用空间、我的空间                        |
| 空间为空            | 常用空间和我的空间都显示空态，引导创建第一个知识空间 |
| 文档为空            | 空间仍可展示，卡片内文档区域显示空态说明             |
| 文档同步中          | 页面保持空间结构，计数或次级文案可显示“同步中”       |
| 创建空间成功        | Notification 绿色 + 刷新空间列表                     |
| 创建空间失败        | 在创建流程内显示错误，不把整页切成错误页             |
| restricted / 无权限 | 不把受限数据伪装成空态；应给出说明与回退路径         |
| 认证错误            | 清除登录状态并跳转 `/login`                          |

## 导航关系

- 从哪来：登录成功后进入 `/`
- 到哪去：
  - 点击空间卡片 → `/spaces/:spaceId`
  - 点击文档入口 → `/documents/:documentId`
  - 退出登录 → `/login`
- 错误回退：认证失败 → `/login`

## 实现约束

- 使用 Mantine 组件：`Paper`, `Stack`, `Group`, `Button`, `Title`, `Text`, `Badge`
- 创建空间入口通过 `CreateSpaceModal` 触发，不在首页常驻展开表单
- 页面数据依赖 `currentUser`, `spaces`, `documents`
- 首页当前不单独维护“全量文档列表面板”
- 首页信息组织优先服务“按空间进入内容”
