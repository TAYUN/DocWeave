# P05 注册页

## 路由
`/register`

## 用途
为新用户提供账号注册入口，并在成功后引导其进入登录或工作台流程。

## 当前阶段定位

- 当前为占位规格，用于避免后续实现完全依赖猜测
- M2 扩展阶段再进入真实实现

## 最小规格

- 主任务：创建账号
- 主内容：邮箱、密码、确认密码、注册按钮
- 状态要求：提交中、校验失败、注册失败、注册成功、disabled、restricted / 无权限
- 导航关系：可从登录页进入，注册成功后返回登录或自动进入工作台
- 实现边界：遵循 Mantine 默认主题，不做营销式注册页

## 状态要求细化

- `disabled`：提交中或表单校验未通过时，注册按钮保持可见但不可操作
- `restricted / 无权限`：若产品采用邀请码或关闭公开注册，应明确说明当前不可注册的原因，并给出返回登录或申请入口

## 后续补全时必须明确

- 是否允许公开注册，还是仅邀请码注册
- 注册成功后的目标页
- 与登录页之间的文案和入口关系
