# 16. 后端安全与校验基线

阅读说明：

1. 本文档负责定义 DocWeave 第一阶段后端安全与请求校验基线。
2. 认证与会话基线以 [14. 认证与会话实现决策](./14.%20认证与会话实现决策.md) 为准。

## 1. 核心结论

第一阶段建议同时纳入：

1. `@adonisjs/shield`
2. `@adonisjs/limiter`
3. `VineJS`

## 2. 为什么要先补这三项

原因如下：

1. 认证、协同 token、编辑器 AI、RAG、文件上传都属于高价值入口。
2. 第一阶段比起追更多功能，更应该先保证接口可信边界清晰。

## 3. 建议基线

### 3.1 Shield

用于：

1. 基础安全头。
2. 常见浏览器安全风险防护。

### 3.2 Limiter

至少保护：

1. 登录接口。
2. 协同 token 接口。
3. 编辑器 AI 接口。
4. RAG 搜索接口。
5. RAG 问答接口。

### 3.3 VineJS

用于：

1. 认证请求。
2. 文档更新请求。
3. 协同 token 请求。
4. 文件上传元数据请求。
5. RAG / AI 输入请求。

## 4. 当前结论

第一阶段后端安全与校验的目标不是“安全体系做满”，而是先让关键入口具备最小可信基线。
