# 25. 图标与视觉标识方案

阅读说明：

1. 本文档负责定义 DocWeave 的图标来源、Logo 使用方向和视觉标识基线。
2. 设计规范总入口以根目录 `DESIGN.md` 为准。
3. UI 页面与壳层视角可继续读 `docs/ui/` 相关文档。

## 1. 文档目的

本文档用于明确 DocWeave 在图标、Logo 和品牌视觉标识上的最小统一规则，避免后续出现页面图标来源混乱、功能图标风格不一致、品牌标识在登录页、工作台、分享页各自跑偏的问题。

## 2. 核心结论

DocWeave 当前视觉标识基线正式确认为：

1. 功能图标统一优先使用 `lucide-react`
2. 品牌标识保持单独设计，不与通用功能图标混用
3. 颜色、字体和品牌语义统一以 `DESIGN.md` 为单一来源
4. 图标只服务于识别和引导，不承担额外视觉噪声

## 3. 为什么采用这条路线

采用该路线的原因如下：

1. `lucide-react` 与当前 React SPA 技术栈集成轻量、维护成本低
2. 图标风格统一对工作台、编辑器周边和设置页非常重要
3. 品牌标识需要独立语义，不能用某个功能图标临时代替
4. 第一阶段应优先建立一致性，而不是先扩大量定制插画或装饰图标

## 4. 功能图标基线

功能图标建议遵循：

1. 优先从 `lucide-react` 选择
2. 同一语义尽量全站复用同一图标
3. 尺寸优先遵循组件层级，而不是单页随意放大缩小
4. 不用多套图标库混搭同一页面

## 5. 品牌标识基线

品牌标识建议至少区分：

1. 主 Logo
2. 简化符号 Mark
3. 文字字标 Wordmark

推荐使用场景如下：

1. 登录页与公开页可用主 Logo 或 Wordmark
2. 侧边栏折叠态、favicon、移动端头部可用简化 Mark
3. 文档页和业务密集区域不应让品牌标识压过任务内容

## 6. 页面使用原则

建议遵循以下页面级原则：

1. 登录页可更明确展示品牌识别
2. 工作台使用克制的品牌露出
3. 文档编辑页弱化品牌，强化任务内容
4. 分享页在只读场景中适当增强品牌可信度

## 7. 图标语义建议

第一阶段建议尽量先统一以下高频语义：

1. 搜索
2. 聊天 / AI
3. 文档
4. 空间
5. 设置
6. 分享
7. 历史版本
8. 收藏
9. 删除 / 归档

## 8. 不建议的做法

需要避免以下问题：

1. 同一功能在不同页面用不同图标表达
2. 品牌 Logo 直接复用通用功能图标
3. 页面里混用多套图标库导致风格割裂
4. 为了“有设计感”过度增加装饰图标噪声

## 9. 第一阶段不重投入的内容

当前不建议第一阶段直接做：

1. 全套自绘功能图标系统
2. 大量品牌插画资产
3. 复杂动态图标语言
4. 多品牌皮肤并存

## 10. 当前结论

DocWeave 第一阶段视觉标识的关键不是“图标多惊艳”，而是先统一三件事：

1. 功能图标来源统一
2. 品牌标识语义独立
3. 设计 token 与品牌色来源统一

只要这三件事先稳定，后续再做更完整品牌系统和视觉升级都不会返工太重。
