# RAG Chat 流式协议设计

## 1. 文档目的

本文档定义 M6 `/api/rag/chat` 的 HTTP streaming 实施边界，避免把业务事件、AI SDK provider 事件和传输格式混在一起。

## 2. Transport 选择

M6 不引入 `socket.io`，也不为 RAG 新增 WebSocket 服务。

1. `/api/rag/search` 使用普通 HTTP JSON。
2. `/api/rag/chat` 使用 HTTP POST 流式响应。
3. 前端使用 `fetch` 消费流，并使用 `AbortController` 取消当前请求。
4. 服务端复用 M5 已验证的 AI SDK/AdonisJS stream adapter。
5. provider 原始事件不得直接暴露给 Web。
6. 流开始前的 HTTP 错误统一使用 `ApiErrorResponse`；流开始后的业务错误使用 `RagStreamError` event。

## 3. Request

```http
POST /api/rag/chat
Content-Type: application/json
Accept: text/event-stream
```

请求体为单轮消息，输入 contract 从 `@docweave/contracts/rag` 导入：

```ts
type RagChatRequest = {
  message: string
  spaceId?: string
  topK?: number
}
```

`spaceId` 只表达用户的体验筛选意图。API 必须先计算当前用户的可见文档范围，再在授权范围内应用该过滤器。HTTP 成功响应和错误响应分别遵循 `ApiSuccessResponse<T>` 与 `ApiErrorResponse`。

## 4. Business event order

业务事件顺序固定为：

```text
start
  -> retrieval
  -> text-delta / citation
  -> finish
```

发生错误时：

```text
start
  -> retrieval (optional)
  -> error
```

失败流不得发送成功的 `finish`。用户主动取消时发送 `finish(cancelled)` 或由 adapter 关闭流，最终选择必须在 execution contract 中与 M5 adapter 行为保持一致。

## 5. Event semantics

- `start`：返回本次请求的临时 `messageId`。
- `retrieval`：返回本次上下文使用的 Citation 集合。
- `text-delta`：返回回答增量文本。
- `citation`：在回答生成过程中补充单个结构化 Citation。
- `finish`：返回结束原因、最终 Citation 集合和 usage。
- `error`：返回稳定的 `RagStreamError`，包含错误码和 retryable 标记。

## 6. Error mapping

| Situation    | HTTP / stream semantic  | Code                    |
| ------------ | ----------------------- | ----------------------- |
| 未登录       | HTTP 401，不启动 stream | `AUTH_UNAUTHORIZED`     |
| space 不可见 | HTTP 403，不查询 Qdrant | `AUTH_FORBIDDEN`        |
| 请求字段非法 | HTTP 422                | `VALIDATION_FAILED`     |
| 检索失败     | stream `error`          | `RAG_RETRIEVAL_FAILED`  |
| 模型调用失败 | stream `error`          | `RAG_GENERATION_FAILED` |
| 流转发失败   | stream `error`          | `RAG_STREAM_FAILED`     |

## 7. Client behavior

1. Web 通过 `apps/web/src/lib/api.ts` 和 feature stream client 消费业务事件，不依赖 provider 原始事件。
2. 发送新请求前取消当前仍在进行的请求。
3. 取消或错误后保留已生成的本地内容，并恢复输入框可用状态。
4. `retrieval` 和 `citation` 只展示经过 API 权限裁剪的来源。
5. Citation 点击后回到当前文档编辑页，无法精确定位时保留文档页并展示非致命提示。

## 8. Contract and adapter boundaries

1. `packages/contracts/src/rag.ts` 定义请求、Citation、搜索结果和 `RagStreamEvent`。
2. `packages/contracts/src/api.ts` 定义 `ApiSuccessResponse`、`ApiErrorResponse` 和共享 HTTP 错误码。
3. `packages/adapters` 负责 Qdrant/AI SDK/legacy payload 到领域 contract 的 shape 转换。
4. `apps/api` 负责权限、状态码和异常映射，不把 HTTP 状态码逻辑放进 adapter。
5. `apps/web/src/lib/api.ts` 负责 transport typing、envelope 解包和稳定错误转换。
6. `apps/web/src/features/rag/lib/rag-view-model.ts` 负责页面展示结构。

## 9. M6 boundary

本协议不包含持久化 conversation、服务端主动推送、多人聊天房间、Socket.IO、导入导出、文件 RAG、发布分享和独立历史 snapshot viewer。
