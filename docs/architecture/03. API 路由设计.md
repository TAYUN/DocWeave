# 03. API 路由设计

阅读说明：

1. 本文档负责定义 DocWeave 第一阶段主业务 API 路由边界。
2. 若需看认证基线，继续读 [14. 认证与会话实现决策](../decisions/14.%20认证与会话实现决策.md)。
3. 若需看搜索与索引状态链路，继续读 [25. 搜索与索引任务编排设计](./25.%20搜索与索引任务编排设计.md)。

## 1. 文档目的

本文档用于回答以下问题：

1. 第一阶段至少需要哪些 API。
2. 哪些接口属于认证、文档、协同、文件、AI、RAG。
3. 哪些接口是第一阶段必须稳定的主链路。

## 2. 路由分组

第一阶段建议按如下分组组织：

```text
/api/auth/*
/api/workspaces/*
/api/spaces/*
/api/documents/*
/api/collaboration/*
/api/files/*
/api/ai/*
/api/rag/*
```

## 3. 认证接口

至少保留：

1. `POST /api/auth/login`
2. `POST /api/auth/logout`
3. `GET /api/auth/me`

## 4. 工作区与空间接口

至少保留：

1. `GET /api/workspaces`
2. `GET /api/spaces`
3. `GET /api/spaces/:spaceId`
4. `GET /api/spaces/:spaceId/tree`

## 5. 文档接口

至少保留：

1. `GET /api/documents/:documentId`
2. `POST /api/documents`
3. `PATCH /api/documents/:documentId`
4. `POST /api/documents/:documentId/snapshots`
5. `GET /api/documents/:documentId/snapshots`
6. `POST /api/documents/:documentId/index`
7. `GET /api/documents/:documentId/index-status`

## 6. 协同接口

至少保留：

1. `POST /api/collaboration/token`

该接口负责：

1. 校验文档权限。
2. 签发短期协同 token。
3. 返回房间名与协同能力。

## 7. 文件接口

至少保留：

1. `POST /api/files/upload`
2. `GET /api/files/:fileId/signed-url`
3. `DELETE /api/files/:fileId`

## 8. AI 接口

第一阶段重点是宿主接入 `@blocknote/xl-ai/server`。

补充约束：

1. 编辑器 AI 接口优先贴近官方协议。
2. 权限与上下文编排仍由 `apps/api` 承担。

## 9. RAG 接口

至少保留：

1. `POST /api/rag/search`
2. `POST /api/rag/chat`

## 10. 第一阶段不做事项

第一阶段不急于补：

1. 复杂分享公开链接 API。
2. 评论与批注 API。
3. 历史会话管理 API。
4. 多种导入格式的复杂批量任务 API。

## 11. 当前结论

第一阶段 API 设计的关键不是“接口多不多”，而是保证以下主链路可用：

1. 认证。
2. 文档读取与更新。
3. 协同 token。
4. 快照与索引状态。
5. 编辑器 AI。
6. RAG 搜索与问答。
