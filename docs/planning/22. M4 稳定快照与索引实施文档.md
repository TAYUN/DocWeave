# 22. M4 稳定快照与索引实施文档

阅读说明：

1. 本文档负责把稳定快照与索引闭环转成可执行实施范围。
2. 快照与索引架构以 [25. 搜索与索引任务编排设计](../architecture/25.%20搜索与索引任务编排设计.md) 为准。

## 1. 阶段目标

让文档内容从协同编辑态进入稳定快照，再进入可追踪的索引任务链路。

## 2. 本阶段包含

1. `document_snapshots`。
2. `snapshotVersion` 语义。
3. `rag_index_jobs`。
4. 数据库轮询 worker。
5. `latestIndexedVersion` 回写。

## 3. 最小任务

1. API 支持显式创建稳定快照。
2. 文档表建立 `latestSnapshotVersion` 与 `latestIndexedVersion`。
3. API 写入索引任务记录。
4. Worker 领取任务并执行到 `Qdrant`。
5. 成功后原子切换已索引版本。

## 4. 完成标准

1. 快照版本和索引版本语义分离成立。
2. 索引任务状态可查询。
3. 失败不破坏上一版已可检索结果。

## 5. 当前阶段补充说明

M4 主线在稳定快照与索引基线之外，后续又单独补了一条协同持久化子线：

1. `feat-m4-collaboration-persistence-baseline`
2. 该子线已完成 closing
3. 当前协同自动持久化只回写 `documents.content`
4. 显式稳定快照与索引动作仍保持独立，不与自动持久化混用

因此，当前 M4 的阶段事实应理解为：

```text
协同运行态
  -> 自动持久化回写 documents.content
  -> 显式创建稳定快照
  -> 显式触发索引任务
```
