# 44. 信息架构字段映射与页面落点

阅读说明：

1. 本文档负责把核心业务对象映射到页面入口和主要展示区域。
2. 信息架构基线以 [10. 信息架构与内容模型设计](../architecture/10.%20信息架构与内容模型设计.md) 为准。
3. 页面清单以 [页面总览地图](../ui/page-inventory.md) 为准。

## 1. 文档目的

本文档用于回答“哪些核心对象在哪些页面出现、页面最少要关心哪些字段”，避免后续对象模型和页面实现逐渐脱节。

## 2. `workspace` 的页面落点

主要页面：

1. 工作台首页
2. 设置页
3. 文档页顶部上下文

最小关注字段：

1. `workspaceId`
2. `name`
3. `currentUserRole`

## 3. `space` 的页面落点

主要页面：

1. 工作台首页
2. 空间详情页
3. 文档页面包屑

最小关注字段：

1. `spaceId`
2. `workspaceId`
3. `name`
4. `role`

## 4. `document` 的页面落点

主要页面：

1. 空间详情页
2. 文档编辑页
3. 搜索结果与 Citation 回跳

最小关注字段：

1. `documentId`
2. `title`
3. `status`
4. `latestSnapshotVersion`
5. `latestIndexedVersion`

## 5. `snapshot` 的页面落点

主要页面：

1. 文档页状态栏
2. 版本历史入口
3. Citation 回跳

最小关注字段：

1. `version`
2. `createdAt`
3. `createdBy`

## 6. `file` 的页面落点

主要页面：

1. 文档页正文中的图片 / 附件
2. 文件上传交互
3. 分享页资源访问

最小关注字段：

1. `fileId`
2. `kind`
3. `name`
4. `mimeType`

## 7. `share` 的页面落点

主要页面：

1. 公开分享页
2. 文档操作区分享入口

最小关注字段：

1. `shareId`
2. `status`
3. `accessLevel`
4. `targetSnapshotVersion`

## 8. 当前结论

信息架构不只是数据库关系，也必须能稳定落到页面。只要对象到页面的落点先清楚，后续 DTO、view-model 和 UI 规格就更容易保持一致。
