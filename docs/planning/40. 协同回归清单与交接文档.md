# 40. 协同回归清单与交接文档

阅读说明：

1. 本文档负责定义 DocWeave 协同闭环的最小回归清单与交接重点。
2. 协同设计与实施分别以 [13. 协同服务设计](../architecture/13.%20协同服务设计.md)、[16. 协同 Token 与 Presence 设计](../architecture/16.%20协同%20Token%20与%20Presence%20设计.md) 和 [20. M3 协同闭环实施文档](./20.%20M3%20协同闭环实施文档.md) 为准。

## 1. 文档目的

本文档用于在协同链路迭代后提供统一回归清单，避免后续每次改完协同 token、Yjs 绑定、Presence 或房间规则后，只做局部验证却漏掉跨浏览器和权限态问题。

## 2. 适用场景

以下改动后应至少跑一轮本文档：

1. `apps/collab` 连接、鉴权或房间逻辑调整
2. `apps/api` 协同 token 接口调整
3. 文档页协同初始化顺序调整
4. Presence / Awareness 字段变更
5. BlockNote 与 Yjs 绑定逻辑调整

## 3. 回归前准备

建议准备：

1. 两个浏览器或两个独立会话窗口
2. 一个可编辑用户
3. 一个只读用户
4. 至少一篇已有正文的测试文档

## 4. 必跑场景

### 4.1 连接建立

1. 可编辑用户打开文档页后可以正常获取协同 token。
2. 文档页可建立 WebSocket 连接。
3. 房间名与目标 `workspaceId + documentId` 匹配。

### 4.2 正文同步

1. 浏览器 A 修改正文后，浏览器 B 可以实时看到变更。
2. 标题与正文都能同步。
3. 连续输入、删除、撤销时不同步不应明显卡死。

### 4.3 Presence

1. 两个在线用户都能看到对方在线。
2. 用户名、头像或颜色展示正确。
3. 只读用户不会被错误展示为可编辑。

### 4.4 权限态

1. 无权限用户无法获取协同 token。
2. 只读用户可进入只读浏览态，但不能编辑正文。
3. 只读用户不会广播可编辑状态。

### 4.5 断线恢复

1. 网络短暂中断后页面可以重连。
2. 重连后房间仍然绑定正确文档。
3. 重连后 Presence 不应长期残留脏状态。

## 5. 重点观察项

回归时重点观察：

1. 是否出现连上了但同步不到内容
2. 是否出现只读用户仍可编辑
3. 是否出现打开 A 文档却连到了 B 房间
4. 是否出现退出后在线状态长时间不消失

## 6. 最小交接信息

每次较大协同改动交接时，建议至少记录：

1. 本次改动涉及哪些协同模块
2. 房间命名是否变化
3. token payload 是否变化
4. Presence 字段是否变化
5. 已跑过哪些手工场景
6. 还有哪些已知限制未覆盖

## 7. 通过口径

协同回归通过的最小标准如下：

1. 两个浏览器可稳定协同同一文档。
2. 只读与可编辑边界正确。
3. Presence 可见且不明显脏乱。
4. 断线恢复后不会误连错房间。

## 8. 当前结论

协同链路最怕“局部看起来能用，联动就漏”。本文档的价值不是增加流程，而是确保协同改动后至少有一套稳定、重复可执行的最小验证口径。

## 9. M4 协同持久化阶段补充记录

适用变更：

1. `feat-m4-collaboration-persistence-baseline`

本次额外关注点：

1. 自动持久化只允许回写 `documents.content`
2. 手动保存才负责创建 stable snapshot
3. 自动持久化不得推进 `latestSnapshotVersion` / `latestIndexedVersion`
4. `onLoadDocument` 必须能从 `documents.content` 恢复正文，而不是依赖浏览器首帧 seed

本次已跑验证：

1. `apps/api` 内部运行态读取/回写接口功能测试通过
2. `apps/collab` 与 `apps/api` 的最小内部握手可用，内部 secret 校验生效
3. 同版本 `@hocuspocus/provider` 双客户端联调通过：同一文档内，客户端 A 写入后客户端 B 可收到变更
4. 自动持久化回写后，`documents.content` 可见最新正文，但 `latestSnapshotVersion` / `latestIndexedVersion` 保持不变
5. 直接预写 `documents.content` 后，新协同连接可通过 `onLoadDocument` 恢复正文

本次暴露并修复的问题：

1. `apps/collab` 本地默认 `API_BASE_URL` 固定为 `127.0.0.1` 时，遇到 Adonis 仅监听 `localhost` / `::1` 会导致内部恢复握手失败，现已改为默认 `http://localhost:3333`
2. 服务端正文恢复 helper 初始化 BlockNote editor 时若不使用目标 blocks，会把默认占位正文带回恢复链路，现已修正为按传入正文初始化

当前仍建议保留的人工观察项：

1. 真浏览器双开下的标题编辑体验
2. 网络短断后的 Presence 清理节奏
3. 只读身份的前端只读展示与广播状态
