# 46. UI 页面规格与实现一致性收口清单

阅读说明：

1. 本文档负责定义页面规格文档、路由结构和实现落地之间的收口检查项。
2. 页面清单以 [页面总览地图](../ui/page-inventory.md) 为准。
3. 路由架构以 `docs/workflow/frontend-route-architecture.md` 为准。

## 1. 文档目的

本文档用于在页面持续迭代后，检查页面规格、路由语义和实际实现是否已经分叉，避免文档说一套、组件和路由实际做另一套。

## 2. 检查重点

建议至少检查：

1. 页面编号是否仍对应正确路由
2. 里程碑状态是否仍准确
3. 页面规格中的关键区块是否已在实现中落位
4. 空态、错误态、只读态约定是否仍一致

## 3. 文档编辑页专项

重点检查：

1. 页面标题区块是否与规格一致
2. 编辑器状态栏是否有落点
3. 协同 / AI / 版本入口是否和文档说明一致

## 4. 搜索与聊天页专项

重点检查：

1. 路由语义是否仍是 `/search` 与 `/chat`
2. 搜索结果、Citation 和跳转行为是否与文档一致
3. 聊天页是否仍遵循轻量自定义 UI 基线

## 5. 当前结论

UI 一致性收口的重点不是挑视觉细节，而是确保页面文档、路由和实现没有逐渐说成三种语言。
