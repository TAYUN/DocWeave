# 41. 索引与搜索回归清单

阅读说明：

1. 本文档负责定义稳定快照、索引任务、搜索和 RAG 检索链路的最小回归清单。
2. 快照、索引与 RAG 设计分别以 [24. 版本与稳定快照设计](../architecture/24.%20版本与稳定快照设计.md)、[25. 搜索与索引任务编排设计](../architecture/25.%20搜索与索引任务编排设计.md)、[26. RAG 与 Citation 设计](../architecture/26.%20RAG%20与%20Citation%20设计.md) 为准。

## 1. 文档目的

本文档用于在索引链路调整后提供统一回归口径，避免后续每次改了快照生成、chunk、embedding 或 Qdrant payload 后，只验证“写进去了”，却漏掉版本一致性和权限过滤。

## 2. 适用场景

以下改动后建议至少跑一轮本文档：

1. 快照生成逻辑调整
2. 索引任务状态流调整
3. chunk 策略调整
4. embedding 模型或参数调整
5. Qdrant payload / filter 逻辑调整
6. 搜索或问答接口返回结构调整

## 3. 必跑场景

### 3.1 快照与任务创建

1. 文档生成新稳定快照后，`snapshotVersion` 正常增加。
2. 创建索引任务后，任务记录包含正确 `documentId` 与 `targetSnapshotVersion`。

### 3.2 任务执行

1. Worker 可以正常领取 `pending` 任务。
2. 任务阶段能按预期推进到 `chunking / embedding / upserting / publishing`。
3. 失败任务不会把当前可用索引版本回退。

### 3.3 版本一致性

1. 成功后 `latestIndexedVersion` 正确更新到目标快照版本。
2. 旧任务成功时不会覆盖更高版本的已发布索引状态。
3. 搜索结果不会混入未激活版本内容。

### 3.4 搜索行为

1. 已索引文档可被搜索命中。
2. 无权限文档不会进入结果。
3. 结果中的 `documentId / snapshotVersion / blockId` 定位信息完整。

### 3.5 RAG 行为

1. 问答可基于当前可用索引版本生成回答。
2. 回答返回的 Citation 可以回到原文。
3. 内容更新并重新索引后，新内容可以替换旧内容被命中。

## 4. 重点观察项

回归时重点观察：

1. 是否出现快照已更新但搜索命中仍是旧内容
2. 是否出现索引失败后上一版结果也不可用
3. 是否出现不同版本 chunk 混查
4. 是否出现权限过滤缺失

## 5. 最小交接信息

每次较大索引 / 搜索改动交接时，建议至少记录：

1. 本次改动涉及哪个阶段
2. chunk 策略是否变化
3. embedding 模型或参数是否变化
4. Qdrant payload / filter 是否变化
5. 已跑过哪些样例文档和问题
6. 已知未覆盖的边界场景

## 6. 通过口径

索引与搜索回归通过的最小标准如下：

1. 快照、索引任务、`latestIndexedVersion` 三层语义一致。
2. 搜索与问答只读取当前有效索引版本。
3. Citation 可以回跳。
4. 失败不会破坏上一版可用结果。

## 7. 当前结论

索引链路最怕“能写进去，但版本和结果不可信”。本文档的作用就是把“写入成功”提升为“版本、权限、检索结果都可信”的最小回归标准。
