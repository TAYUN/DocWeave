# 10. 第一阶段验收标准与测试策略

阅读说明：

1. 本文档负责回答“第一阶段每个里程碑怎么算完成、怎么验证”。
2. 当前阶段顺序以 [00. 当前实施路线图](./00.%20当前实施路线图.md) 为准。
3. 若需看当前技术拍板，以 [01. 最终技术方案确认](../decisions/01.%20最终技术方案确认.md) 为准。

## 1. 文档目的

本文档用于明确第一阶段各里程碑的验收口径、测试层次和最小验证方法，避免后续出现“功能看起来差不多，但没人能说清是否闭环”的情况。

## 2. 核心结论

第一阶段验收不以“做了多少页面”或“装了多少技术栈”为准，而以以下五条链路是否闭环为准：

1. 业务元数据链路。
2. 文档编辑链路。
3. 协同链路。
4. 稳定快照与索引链路。
5. 编辑器 AI 与 RAG 问答链路。

## 3. 测试层次

第一阶段建议至少分为四层测试：

1. `typecheck`
2. `unit / contract test`
3. `integration test`
4. `manual scenario verification`

### 3.1 typecheck

用于保证：

1. app / package 边界未破坏。
2. contract 结构未失配。
3. 关键输入输出类型仍然一致。

### 3.2 unit / contract test

用于保证：

1. 纯函数逻辑正确。
2. schema / contract 结构稳定。
3. chunk builder、prompt builder、permission checker 等核心逻辑可回归。

### 3.3 integration test

用于保证：

1. API 与 service 之间的联动成立。
2. 数据库、权限、快照、索引状态等跨层行为正确。
3. HTTP 和流式协议没有断。

### 3.4 manual scenario verification

用于保证：

1. 多浏览器协同体验成立。
2. 编辑器 AI 结果能正确回写。
3. 搜索与聊天页面行为符合预期。
4. 权限和错误态在真实页面中可理解。

## 4. M1 验收标准

范围：

1. Monorepo 基础工程。
2. 四端应用骨架。
3. 基础依赖联通。

最低通过标准：

1. `apps/web`、`apps/api`、`apps/collab`、`apps/worker` 可以分别启动。
2. `packages/*` 基础包可以通过 `typecheck`。
3. `PostgreSQL`、`Redis`、`Qdrant`、`MinIO` 本地可联通。
4. 环境变量模板完整。

## 5. M2 验收标准

范围：

1. 认证与会话。
2. 工作区、空间、文档元数据。
3. 单人编辑器接入。

最低通过标准：

1. 用户可以登录、登出、读取当前用户。
2. 工作区、空间、文档基础 API 可用。
3. 前端可展示文档树。
4. 文档页可进入。
5. BlockNote 单人编辑可用。

## 6. M3 验收标准

范围：

1. 协同最小闭环。

最低通过标准：

1. 两个浏览器可同时打开同一文档并同步正文。
2. 标题和正文都可同步。
3. Presence 基础可见。
4. 协同 token 接口语义成立。

## 7. M4 验收标准

范围：

1. 稳定快照与索引基线。

最低通过标准：

1. 文档可以生成稳定快照。
2. `snapshotVersion` 语义成立。
3. Worker 可以从数据库任务记录中领取索引任务。
4. 索引任务能完成 `snapshot -> chunk -> embedding -> qdrant`。
5. `latestIndexedVersion` 能正确回写。

## 8. M5 验收标准

范围：

1. 编辑器 AI 最小闭环。

最低通过标准：

1. 至少支持选区入口和 Slash 入口。
2. 至少支持改写、扩写、缩写、翻译、总结五个动作。
3. AI 结果支持流式返回。
4. 用户可以应用或放弃结果。
5. 只读状态下禁止应用结果。

## 9. M6 验收标准

范围：

1. RAG 搜索与问答最小闭环。

最低通过标准：

1. 已索引文档可被搜索。
2. `POST /api/rag/chat` 可流式返回回答。
3. 回答可返回最小 Citation。
4. Citation 可回到 `documentId + snapshotVersion + blockId`。

## 10. 当前结论

第一阶段的验收标准不是“文档写完”或“基础设施都装了”，而是每条主链路都能明确回答：

1. 这条链路通没通。
2. 关键协议稳不稳。
3. 页面实际体验能不能用。
