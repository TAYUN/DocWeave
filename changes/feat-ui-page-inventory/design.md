# UI 页面清单设计方案

## 目标

为 DocWeave 建立一套"页面规格"文档层，填补 DESIGN.md（视觉规范）与实际代码之间的缺口。让 AI 实现页面时不再靠猜测，而是有明确的文本级规格可循。

## 设计原则

### 1. 页面规格的定位

```text
DESIGN.md（视觉气质：稳定、清晰、低噪音的工作台）
    ↓
实现指南（Mantine 默认主题实现规则：怎么配、怎么写）
    ↓
UI 页面规格 ⬅️ 本方案要补的这一层（每页的用途、布局、状态、导航）
    ↓
实际页面代码
```

### 2. 参考项目的设计方法

借鉴 `ai-doc-flow-v0` 的设计思路，其方法可以提炼为：

```
1. 全局布局框架先行
   - 定义 AppHeader + AppSidebar + MainContent 的固定结构
   - 定义 Sidebar 的多种模式（Global/Space/Settings/AI）
   - 定义关键尺寸和布局行为

2. 每页一个独立规格
   - ASCII 线框：用纯文本画出页面布局
   - 内容分区：定义每个区块的任务
   - 状态设计：loading / empty / error / not found 各是什么样
   - 禁用与受限：区分 disabled 和 restricted，不把“不可用”混成一个桶
   - 导航关系：从哪来、到哪去

3. 跨页面一致性问题统一兜底
   - 空态设计风格统一
   - 操作反馈风格统一
   - 加载状态规范统一
   - 权限降级策略统一

4. 优先级标注
   - 每个页面标实现优先级
   - 标注依赖关系（先有什么才能有这个页面）
```

### 3. DocWeave 适配规则

引用 ai-doc-flow-v0 的设计方法，但不照搬其页面结构和组件体系：

| 层面 | ai-doc-flow-v0 做法 | DocWeave 做法 |
|------|-------------------|--------------|
| 组件库 | shadcn/ui + Tailwind v4 | Mantine 默认主题 |
| 路由 | Next.js App Router / React Router | TanStack Router |
| 颜色体系 | 自定义 CSS 变量（oklch） | Mantine 默认主题色 + 语义色 |
| 布局 | Tailwind CSS 类 | Mantine 组件 + 少量 CSS |
| Sidebar 模式 | React state + 路由判断 | TanStack Router 路由匹配 + 状态 |
| 字体 | Inter + JetBrains Mono | Mantine 默认字体栈 |
| 暗色模式 | 手动 .dark class 切换 | Mantine 内置 colorScheme 切换 |

### 4. 页面规格文件结构

每个页面规格文件统一包含以下章节：

```markdown
# 页面名称

## 路由
`/path/:param`

## 用途
一句话说清这个页面解决什么问题。

## 布局（ASCII 线框）
用纯文本线框画出页面结构。

## 内容分区
描述每个区块的任务和内容类型。

## 状态规格

| 状态 | 表现 |
|------|------|
| loading | 骨架屏 / 加载指示器 |
| empty | 空态提示 + 下一步动作 |
| error | 错误说明 + 重试/回退 |
| not found | 对象不存在 |
| disabled | 说明当前不能操作的原因，并保持控件可辨认 |
| restricted / 无权限 | 说明为什么不能访问，以及用户还能做什么 |

## 导航关系
- 从哪来
- 到哪去

## 实现约束
- 涉及哪些组件
- 使用哪些 API
- 对接哪些数据
- Mantine 实现边界说明
```

## 全局布局框架设计

### 当前结构

DocWeave 当前使用 `WorkbenchShell` 组件作为已登录页面的外壳：

```plaintext
┌──────────────────────────────────────────────┐
│  Sidebar (aside.sidebar)                     │
│  ┌────────────────────────────────────────┐  │
│  │ DocWeaver 品牌标识                     │  │
│  │ 当前用户卡片（姓名/邮箱/退出）          │  │
│  │ 导航按钮：工作台                       │  │
│  │ ────────────────────────────────────  │  │
│  │ 空间树                                │  │
│  │ ────────────────────────────────────  │  │
│  │ 当前路由信息（调试用途）               │  │
│  └────────────────────────────────────────┘  │
├──────────────────────────────────────────────┤
│  MainContent (<Outlet />)                    │
│  ┌────────────────────────────────────────┐  │
│  │  路由页面内容渲染区                     │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

### 目标布局框架（M2 目标）

```plaintext
┌──────────────────────────────────────────────────┐
│  AppHeader（固定高度 h-11，横向布局）              │
│  [Logo] [品牌名]    [全局搜索 ⌘K]    [新建][通知][头像▼] │
├──────────┬───────────────────────────────────────┤
│          │                                       │
│  Sidebar │  MainContent                          │
│  (w-56)  │  (flex-1, overflow-auto)              │
│          │                                       │
│  ★ 根据  │  <页面内容渲染区>                      │
│  当前路由 │                                       │
│  自动切换 │                                       │
│  显示内容 │                                       │
│          │                                       │
└──────────┴───────────────────────────────────────┘
```

### AppHeader 设计约束

- AppHeader 是工作台的全局操作区，不是品牌展示位
- M2 阶段 Header 只承载当前阶段能闭环的全局动作：
  - 返回工作台首页
  - 新建入口（至少支持创建空间或进入当前已定义的新建流程）
  - 当前用户菜单 / 退出登录
- 全局搜索可以作为占位入口出现，但在 M7 搜索能力落地前必须标注为“未启用”或“即将提供”，不能伪装成可用功能
- 通知入口不属于当前 change 的必备能力；若保留占位，必须在文档中明确为未来扩展，不应默认进入实现必做项
- Header 中每个入口都应说明点击后的去向、不可用态和回退路径，避免后续 AI 自由塞入无闭环按钮

### 响应式约束

- Desktop：保留 Header + Sidebar + MainContent 三段结构，主任务内容始终优先占据视觉中心
- Tablet：Sidebar 可收窄或切换为抽屉，但当前页面主任务区不得被顶部全局操作挤压
- Mobile：单列优先，Sidebar 转为抽屉或二级入口；次级信息（元数据、辅助导航、扩展操作）后置、折叠或下沉
- 所有页面规格在响应式下都要先保护主任务，再安排次级信息，不能把桌面并列区块机械堆成单列

### Sidebar 模式定义

| 模式 | 触发路由 | 显示内容 |
|------|---------|---------|
| GlobalMode | `/` 工作台首页 | 快捷导航 + 空间列表入口 + 最近文档 |
| SpaceMode | `/spaces/:spaceId` | 空间切换器 + 空间内文档树 |
| DocumentMode | `/documents/:documentId` | 空间面包屑 + 文档树 + 文档操作 |
| 无 Sidebar | `/login` 等公开页 | 不显示 |

## 跨页面一致性约定

| 问题 | 约定 |
|------|------|
| 空态设计 | 统一说明"还没有内容" + 给出下一步动作按钮 |
| 错误反馈 | 使用 Mantine Alert 组件，红色语义 |
| 加载状态 | 首次加载用骨架屏 / Paper 占位块，局部操作用 Loader |
| 操作反馈 | 使用 Mantine Notification 组件 |
| not found | 统一错误卡片："未找到对应XX" + 返回/重试 |
| disabled | 控件保持可见但不可操作，并明确说明当前不能执行的原因 |
| restricted / 无权限 | 页面级无权限显示明确说明与回退路径；组件级无权限优先提供说明或替代路径，而不是一律隐藏 |
